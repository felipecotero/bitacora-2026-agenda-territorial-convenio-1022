Conectar el Telar con GitHub

Esta guía es para Mercy, Jeimmy y Catalina. Sirve para que los cambios que hagan en el Telar (mover una sesión, agregar una nueva, cambiar un horario) se guarden automáticamente en el repositorio del proyecto, sin necesidad de bajar archivos ni subirlos a mano.

1

Asegurarse de tener cuenta en GitHub

Si aún no tienes cuenta, créala gratis en github.com/signup. Usa preferiblemente tu correo institucional. Anota tu nombre de usuario; lo vas a necesitar más adelante.

Si ya tienes cuenta, salta al paso 2.
2

Pedirle al administrador del repo acceso de escritura

El repositorio del Telar lo administra una sola persona del equipo (probablemente Catalina). Esa persona debe agregarte como colaboradora:

Repositorio › Settings › Collaborators › Add people

Vas a recibir una invitación por correo. Acéptala antes de seguir.

3

Crear tu token personal

El token es una llave que le das al Telar para escribir en GitHub a tu nombre. Tú la creas, tú la borras. Pasos:

  1. Entra a github.com/settings/tokens?type=beta
  2. Clic en Generate new token (botón verde arriba a la derecha)
  3. En Token name escribe algo claro: Telar – Mercy
  4. En Expiration elige Custom y pon una fecha 90 días adelante (después puedes renovarla)
  5. En Repository access elige Only select repositories y selecciona el repo del Telar
  6. En Repository permissions baja hasta Contents y cámbialo a Read and write
  7. Baja al final y clic en Generate token
  8. Te aparece un código tipo github_pat_11AAA… — cópialo ya, GitHub no te lo va a mostrar de nuevo
Importante: nunca compartas el token por chat, correo o WhatsApp. Si crees que alguien lo vio, vuelve a la página de tokens y bórralo (Revoke). Crear uno nuevo toma 2 minutos.
4

Pegar el token en el Telar

Vuelve a esta agenda, abre la sección El Telar y arriba verás un botón gris que dice Configurar GitHub. Clic ahí.

Llena los campos con:

  • Token: el código github_pat_… que copiaste
  • Owner: el usuario u organización donde vive el repo (ej. mincultura o el usuario de Catalina)
  • Repositorio: el nombre del repo (ej. agenda-territorial)
  • Rama: deja main a menos que el equipo use otra
  • Ruta al archivo: deja data.json
  • (Opcional) tu nombre y correo, para que los commits queden a tu nombre

Clic en Probar conexión. Si todo está bien, aparece un mensaje verde. Luego clic en Guardar configuración.

✓ Listo. El badge en la esquina superior del Telar pasa de gris (Configurar GitHub) a verde (conectado).
5

Editar normalmente

A partir de aquí no hay que pensar más en GitHub. Cada vez que cambies algo:

  • El badge cambia a amarillo: Guardando…
  • A los 1-2 segundos pasa a verde: ✓ Guardado en GitHub
  • En el repo aparece un commit nuevo a tu nombre, con la hora del cambio

Si algo falla (sin internet, token vencido) el badge se pone rojo y dice Error al guardar. Tus cambios siguen guardados localmente, así que no se pierden — solo arregla la conexión y vuelve a editar cualquier cosita para forzar el guardado.

Checklist rápida

  • Tengo cuenta en GitHub y soy colaboradora del repo
  • Generé mi token personal con permiso Contents: Read and write
  • Copié el token en un lugar seguro (gestor de contraseñas o nota cifrada)
  • Pegué el token en el Telar y la prueba de conexión salió verde
  • Hice un cambio de prueba y vi el commit aparecer en GitHub
